# 지점 스케줄 자동 수집 — 설정 안내 (비개발자용)

세 지점(바디코디·스튜디오메이트·버핏)의 스케줄을 **자동으로 수집**해서
대시보드(**https://bodynox.netlify.app/dashboard/**)에 보여주는 시스템입니다.

## 어떻게 동작하나요?
1. GitHub의 자동 로봇(Actions)이 **2시간마다** 세 사이트에 로그인해 스케줄을 읽어옵니다.
2. 읽어온 스케줄을 `dashboard/data/schedule.json` 에 저장(변동이 있을 때만)합니다.
3. 대시보드는 그 파일을 읽어 세 지점을 한 화면에 보여줍니다. **새로고침**을 누르면 최신본을 즉시 다시 불러옵니다.
4. **즉시 수집이 필요할 때**: GitHub 저장소 → **Actions** 탭 → "지점 스케줄 자동 수집" → **Run workflow** 를 누르면 그 자리에서 수집합니다(약 1~2분).

> ⏱ **왜 2시간 간격?** 이 저장소는 비공개라 GitHub 무료 자동화 시간이 월 한도(2000분)가 있습니다.
> 2시간 간격이면 한도 안에서 안전합니다. 더 자주 원하시면 간격을 줄일 수 있지만(유료 한도 주의),
> 보통은 2시간 자동 + 필요할 때 수동 즉시 수집으로 충분합니다.

## ✅ 회원님이 한 번만 해주실 설정 (제가 화면을 열어 안내합니다)

### 1) 세 지점 로그인 정보 등록 (GitHub Secrets)
자동 로봇이 로그인할 수 있도록, 로그인 정보를 **암호화된 금고(Secrets)** 에 넣습니다.
저장소 → **Settings → Secrets and variables → Actions → New repository secret** 에서
아래 이름으로 각각 추가합니다. (값은 회원님만 입력, 저는 값을 보지 못합니다.)

| Secret 이름 | 값 |
|---|---|
| `BODYCODI_USER` | 바디코디 로그인 아이디(이메일) |
| `BODYCODI_PASS` | 바디코디 비밀번호 |
| `BODYCODI_VIEW_PASS` | (있으면) 내부 지점만 보기 비밀번호 |
| `STUDIOMATE_USER` | 스튜디오메이트 로그인(휴대폰번호) |
| `STUDIOMATE_PASS` | 스튜디오메이트 비밀번호 |
| `BUTFIT_USER` | 버핏 로그인(휴대폰번호) |
| `BUTFIT_PASS` | 버핏 비밀번호 |

### 2) 대시보드 비밀번호 잠금 (Netlify)
내부 스케줄이 아무나 보이지 않도록 대시보드에 자물쇠를 겁니다.
Netlify → 사이트 → **Site configuration → Environment variables → Add** 에서
- 이름: `DASH_PASSWORD`
- 값: 원하는 잠금 비밀번호

를 넣고 저장하면, 다음 배포부터 `/dashboard/` 접속 시 아이디 **`bodynox`** + 그 비밀번호를 물어봅니다.
(비밀번호를 설정하기 전까지는 잠기지 않아, 설정 중 잠금 사고가 나지 않습니다.)

---

## 🔒 보안 메모
- 로그인 정보는 **Secrets/환경변수에만** 들어가며 코드·기록에 남지 않습니다.
- 채팅으로 공유하신 비밀번호는 기록에 남을 수 있으니, 가능하면 **새 비밀번호로 변경 후** 위 금고에 넣어 주세요.
